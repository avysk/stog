Release 0.6 of stog is available from the page.

Changes:

  • mod: do not write the config file if it already exists,
  • add: <documents> node handles reverse="false" attribute not to reverse order of documents (which is default, i.e. the more recent first),
  • mod: better cache system, using dependencies of each document on other documents and files; also handles the stog environment used to compute documents in cache, so that a change in the stog environment will make all documents recomputed (for example when language changes),
  • add: a rule using a file or another document must add a dependency from the computed document to the document or file it depends on, using Stog_plug.add_dep,
  • add: <include> now handles a depend="false" attribute, to prevent adding a dependency of the document on the included file,
  • add: --depcut option, used when getting cached documents, to use only one level of dependency between documents; this is useful for example in case the site has a menubar, leading to having almost all documents depending on almost all documents.
